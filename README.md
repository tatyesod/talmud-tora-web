# מערכת ניהול תלמוד תורה - גרסת Web (v2 - מסד הנתונים האמיתי)

המרה מלאה של מערכת ה-Access (frontend + backend) לאפליקציית Web: Node.js + Express + SQLite,
בעברית, RTL.

## מה תוקן מהגרסה הקודמת

הגרסה הראשונה יובאה מקובץ frontend בלבד (שהכיל טבלאות זמניות/מטמון של 449 תלמידים פעילים
בלבד). לאחר שאיתרנו ויובא קובץ ה-**Backend** האמיתי (`תלמוד תורה החדש מסד נתונים.accdb`),
המערכת עודכנה לחלוטין:

- **1,431 תלמידים** (לעומת 449 קודם) — כולל **כל 3 הסטטוסים**: פעיל (837), ארכיון (358), לא פעיל (233)
- **647 משפחות** מנוהלות כטבלה נפרדת ומקושרת (לא משוכפלות בכל תלמיד)
- **75 מלמדים/עובדים**, עם שיבוץ לכיתות (טבלת קישור `מלמדים בכתות`)
- **32 כיתות** עם קישור לקטגוריית שכר לימוד
- **21 מחזורים**, **5 קטגוריות שכר לימוד** (עם תמחור), **2 סוגי הנחות**
- **1,229 סבים/סבתות**, **60 ישיבות**, **15 חסידויות**
- **תיק תלמיד** (יומן הערות/מעברי כיתה) - 17 רשומות
- **697 אנשי קשר לשעת חירום** (מקושרים למשפחה, לא לתלמיד בודד)

כל הטבלאות מקושרות כראוי (Foreign Keys) במקום שכפול נתונים כמו בגרסה הקודמת.

## פיצ'רים

- **תלמידים**: רשימה/חיפוש/סינון (כיתה, מחזור, סטטוס), צפייה, הוספה/עריכה/מחיקה,
  קישור למשפחה/אחים/מלמדי הכיתה, תיק תלמיד (יומן הערות מתעדכן), הדפסת טופס בריאות
  ואנשי קשר לחירום
- **מלמדים ועובדים**: רשימה/חיפוש, צפייה, הוספה/עריכה/מחיקה, כיתות בהוראה
- **משפחות**: כרטיס משפחה מרוכז עם כל הילדים ואנשי הקשר לחירום
- **כיתות ומחזורים**: רשימת כיתות עם תלמידים פעילים ומלמדים, רשימת מחזורים
- **שכר לימוד**: טבלת קטגוריות תמחור + טבלת הנחות לפי מספר אחים
- המרת תאריכים לועזי↔עברי מדויקת (נבדקה ואומתה מול 73,000+ רשומות מקור)

## עדכון אמינות: בנייה אוטומטית של מסד הנתונים (תוקן קריסת "database disk image is malformed")

בגרסה זו תוקנה בעיה שבה קובץ מסד הנתונים הבינארי (`talmud-tora.db`) היה עלול
**להיפגם בהעברה** (הורדה מהדפדפן, חילוץ ZIP, אנטי-וירוס, סנכרון ענן על תיקיית
Downloads) ולגרום לשגיאת `database disk image is malformed` בהרצה.

**הפתרון**: המערכת כעת לא שולחת את קובץ ה-`.db` הבינארי בכלל. במקום זאת, כל
הנתונים שמורים בקובץ טקסט (`server/data/seed.json`) שעמיד הרבה יותר להעברה.
**בכל הפעלה ראשונה** (`npm start`), אם קובץ ה-`.db` חסר — או אם הוא קיים אבל
פגום (נכשל בבדיקת תקינות) — המערכת **בונה אותו אוטומטית מחדש** מתוך `seed.json`,
ללא שום פעולה ידנית נדרשת.

זה אומר שאם אי פעם תקבלו שוב שגיאת מסד נתונים פגום, פשוט מריצים `npm start`
שוב (או מוחקים את `server/data/talmud-tora.db` ומריצים `npm start`) וזה יתקן
את עצמו אוטומטית.

## הרצה מקומית

```bash
cd app
npm install
cp .env.example .env
node server/seedUsers.js   # יוצר 4 משתמשי ברירת מחדל (פעם ראשונה בלבד)
npm start
```

האפליקציה תרוץ ב-http://localhost:3000 — ראו פרטי משתמשי ברירת המחדל בסעיף
"עדכוני הגרסה האחרונה" למטה (**חובה לשנות סיסמאות אחרי הכניסה הראשונה**)

```bash
export LANG=C.utf8 LC_ALL=C.utf8

```

## פריסה אונליין (Production)

1. **Render.com / Railway.app** — לחבר ל-GitHub, להגדיר `npm start`, להוסיף Persistent
   Disk שמצביע ל-`server/data/` (כדי שמסד הנתונים יישמר בין דיפלוים)
2. **VPS** — Node 22+, `pm2` להרצה רציפה, Nginx + SSL (Let's Encrypt)
3. **שרת מקומי קיים** — `npm start` + גישה מרחוק/VPN

## ריבוי משתמשים בו-זמנית

המערכת תומכת במספר משתמשים הנכנסים יחד באותו הזמן, ללא התנגשויות:
- כל הבקשות עוברות דרך אותו שרת Node.js יחיד עם חיבור מסד נתונים משותף — אין שמירה
  מקומית (cache) בדפדפן, כך שכל פעולת רענון/שמירה מציגה מיד את הנתון העדכני ביותר
  שכל משתמש אחר שמר
- מסד הנתונים (SQLite) מוגדר במצב **WAL** (Write-Ahead Logging) המאפשר קריאה וכתיבה
  בו-זמנית ממספר משתמשים ללא נעילות מיותרות
- מתאים לשימוש של כמה עשרות משתמשים בו-זמנית (לדוגמה: מזכירות, מנהל, כמה מורים).
  לעומס גבוה משמעותית יותר (מאות משתמשים בו-זמנית) מומלץ בעתיד לשקול מעבר ל-PostgreSQL

## ניהול שנת לימודים והעלאת שנה

בתפריט "ניהול שנה" (וגם בכותרת דף השער):
- מוצגת שנת הלימודים העברית הנוכחית (לדוגמה תשפ"ו), המחושבת אוטומטית מתאריך היום
  בהפעלה הראשונה
- **העלאת שנה**: פעולה אחת שמבצעת בו-זמנית:
  1. שמירת תמונת מצב מלאה (Snapshot) של השנה הנוכחית, הניתנת לצפייה בכל עת תחת
     "שנים קודמות"
  2. קידום אוטומטי של כל התלמידים המשובצים לכיתה: כיתה א'→כיתה ב' וכן הלאה, לפי
     התאמת מקבילה (אם יש כיתה תואמת פעילה; אחרת נבחרת המקבילה הראשונה הפעילה)
  3. תלמידי כיתה ח' (הכיתה האחרונה) עוברים אוטומטית לסטטוס "ארכיון"
  4. תלמידים מכיתות "עדיין לא נכנסו" (לפי מספר המקבילה) עוברים למכינה א' התואמת
     והופכים לסטטוס "פעיל"
  5. תלמידים ללא שיבוץ לכיתה (אחים קטנים שטרם הגיעו לגיל כניסה) **אינם משתנים** —
     נשארים ללא כיתה, בסטטוס "לא פעיל", כפי שהיו
  6. השנה העברית במערכת מתקדמת בשנה אחת

הערה: ההתאמה בין כיתה לכיתת היעד מבוססת על **שם הכיתה ומספר המקבילה** (לדוגמה
"כיתה ג' (2)" → "כיתה ד' (2)"). אם בעתיד ירצו לשנות את כללי הקידום (מיזוג/פיצול
מקבילות, מספר כיתות שונה משנה לשנה), הלוגיקה נמצאת במלואה ב-`server/yearManager.js`
וניתנת להתאמה.

- **שכפול הדוחות המקוריים** מ-Access: "יומן" (8 עמודים), "רישום", "מדבקות למשפחות",
  "דוח משפחות" — קיימים כ-69 טפסים ו-25 דוחות במקור, אבל אלו אובייקטים גרפיים פנימיים
  של Access ש-mdbtools לא יכול לחלץ; כדי לשחזר אותם נדרשת צפייה בכל דוח (צילום מסך/PDF)
  ובנייתו מחדש כתבנית הדפסה ב-HTML
- **חישוב שכר לימוד אוטומטי** לפי הנחות/אחים (הנתונים קיימים, אבל הלוגיקה לחישוב
  סכום סופי למשפחה לא נבנתה עדיין)
- ריבוי משתמשים עם הרשאות נפרדות (מנהל/מורה/מזכירה)
- ייצוא לאקסל/PDF
- היסטוריית שינויים (audit log)

## עמידות מסד הנתונים (הגנה מפני קבצים פגומים)

קובץ `talmud-tora.db` הוא קובץ בינארי שיכול להיפגם בהעברה (zip, אנטי-וירוס, ענן).
לכן המערכת שומרת גם עותק טקסטואלי מלא (`server/data/seed.json`) שממנו ניתן לבנות
מחדש את כל מסד הנתונים תוך פחות משנייה:

- **בכל הפעלת שרת**: נבדקת תקינות מסד הנתונים (`PRAGMA integrity_check`). אם הקובץ
  פגום (או חסר), השרת **בונה אותו מחדש אוטומטית** מתוך `seed.json`, ללא צורך בהתערבות
- **סגירה מבוקרת**: לחיצה על Ctrl+C עוצרת את השרת בצורה נקייה ומוודאת שקבצי ה-WAL
  מתאחדים בחזרה לקובץ הראשי (מונע בעיות בפעם הבאה)
- אם אי פעם תרצו לעדכן את `seed.json` הידני אחרי שינויי נתונים משמעותיים, ניתן להריץ:
  ```bash
  node server/exportSeed.js
  ```
  זה ייצא את המצב הנוכחי המלא של מסד הנתונים ל-`seed.json` (שיוכל לשמש לשחזור עתידי)

## עדכוני הגרסה האחרונה

- **ניהול כוח אדם**: בכרטיס מלמד נוסף מעקב נוכחות/חיסורים (נוכח/נעדר/חופשה/מחלה,
  עם סיכום כמותי) ו"תיק עובד" (חוזה/תעודה/הערכה/אחר) — דומה למבנה "תיק תלמיד"
- **תקשורת הורים במייל**: עמוד "תקשורת הורים" עם שני חלקים:
  1. שליחת מייל קבוצתי — בחירת כיתה (או כל ההורים), המערכת אוספת את כל כתובות
     המייל (אב+אם) ופותחת חלון Gmail עם כולם ב-**BCC** (כדי שהורים לא יראו אחד
     את כתובת השני)
  2. **פניות הורים** — רישום/מעקב פניות (נושא, תוכן, סטטוס: פתוח/בטיפול/סגור,
     ומענה)
- **אירועים ולוח שנה**: עמוד "אירועים" עם רשימת אירועים עתידיים/עבר (טיולים, חגים,
  אסיפות), כל אירוע יכול לכלול מחיר ודרישת הרשמה, עם ניהול נרשמים (תלמיד + סימון
  שולם/לא שולם)
- **מלאי ותחזוקה**: רשימת ציוד לפי כיתה/מיקום עם כמות ומצב, ובנפרד **בקשות תחזוקה**
  (תקלה/דיווח, עם סטטוס פתוח/בטיפול/סגור ותיעוד מי דיווח)
- **הוצאות מול ספקים**: כל הוצאה מקושרת לספק (מהמודול שכבר היה קיים), עם תאריך,
  סכום, קטגוריה, מספר חשבונית וסימון שולם/לא שולם. מוצג גם סיכום כולל לפי ספק,
  וגם בכרטיס הספק עצמו מופיעה היסטוריית ההוצאות וסך הכל ששולם לו

- **ריבוי משתמשים אמיתי** (4 משתמשים, ניתן להוסיף עוד): כל משתמש מתחבר בנפרד עם שם
  משתמש וסיסמה משלו (לא יותר משתמש יחיד דרך משתני סביבה). משתמשי ברירת מחדל לאחר
  הרצת `node server/seedUsers.js`:
  - `admin` / `admin123` (מנהל המערכת)
  - `user2` / `user2123`
  - `user3` / `user3123`
  - `user4` / `user4123`

  **חשוב מאוד: שנו את כל הסיסמאות האלו** מיד אחרי הכניסה הראשונה, דרך "👥 משתמשים"
  בתפריט → עריכה → סיסמה חדשה.
- **דף בית אישי לכל משתמש**: רשימת "המשימות שלי" (הוספה/סימון כבוצע/מחיקה, עם תאריך
  יעד אופציונלי), ופאנל "משתמשים מחוברים" שמראה בזמן אמת (מתעדכן כל 15 שניות) מי
  מהמשתמשים נמצא כרגע במערכת (נקודה ירוקה = פעיל ב-90 השניות האחרונות)
- **הודעות בין משתמשים**: לחיצה על "💬 הודעות" בתפריט פותחת תיבת דואר פנימית — שיחה
  נפרדת עם כל משתמש אחר, עם סימון הודעות שלא נקראו
- **שדה "מספר ילדים בבית" למלמד**, לצורך תכנון תקציב חלוקת מתנות — מופיע בכרטיס/טופס
  המלמד, ניתן למיין לפיו ברשימת המלמדים, ומוצג סכום כולל בראש הרשימה
- **רשימת תלמידים מורחבת**: העמוד עכשיו ברוחב מלא (לא מוגבל ל-1100px), עם עמודות
  נוספות: מגזר, סניף, שם האב, נייד אב, שם האם, נייד אם, עיר
- **דוחות אקסל**: כל הדוחות מיוצאים כעת מימין לשמאל (RTL אמיתי בקובץ ה-Excel, לא רק
  טקסט בעברית בכיוון LTR), ממוינים לפי כיתה (כשרלוונטי), עם כותרת מעוצבת בראש כל
  גיליון (שם המוסד + שם הדוח + תאריך הפקה) ולוגו קטן בפינה הימנית העליונה. שדרגנו
  לספריית `exceljs` לצורך זה (התומכת בעיצוב ותמונות, בניגוד לספריה הקודמת)

- **זמני היום ההלכתיים**: בלחיצה על "זמני היום ⌃" ברצועת המידע התחתונה נפתחת רשימה
  מלאה: עלות השחר, הנץ החמה, סוף זמן ק"ש (גר"א ומג"א), סוף זמן תפילה (גר"א ומג"א),
  חצות היום, מנחה גדולה, מנחה קטנה, פלג המנחה, שקיעה, צאת הכוכבים, וחצות הלילה —
  הכל מ-Hebcal Zmanim API, מחושב לבני ברק לפי תאריך היום

- **רצועת מידע בתחתית דף השער**: שעון חי, תאריך עברי ויום בשבוע (מחושב מקומית
  במדויק על ידי המערכת), דף יומי, מזג אוויר נוכחי בבני ברק, וזמני כניסת/יציאת שבת —
  כל אלו נשלפים מ-2 שירותים חיצוניים חינמיים שאינם דורשים מפתח API:
  [Hebcal](https://www.hebcal.com/home/developer-apis) (לוח עברי, דף יומי, זמני שבת)
  ו-[Open-Meteo](https://open-meteo.com) (מזג אוויר). הנתונים נטענים ישירות בדפדפן
  (לא דרך השרת), כך שאין השפעה על ביצועי השרת. **דורש חיבור אינטרנט פעיל** במחשב
  בו נצפה הדף (לא בהכרח באותו שרת שמריץ את המערכת)

- **שיבוץ מלמד לכיתה מתוך עריכת מלמד**: בעמוד עריכת מלמד יש כעת קטע "שיבוץ לכיתות" —
  ניתן להוסיף שיבוץ (כיתה + תפקיד: בוקר/אחה"צ/עוזר) או להסיר שיבוץ קיים. השינוי
  משתקף מיד בכרטיס הכיתה, בכרטיס המלמד, ובכרטיסי כל התלמידים באותה כיתה
- **מחיקת משפחה**: בכרטיס משפחה נוסף כפתור "מחיקת משפחה". מחיקה אינה מוחקת את
  התלמידים המשויכים — היא רק מסירה את הקישור שלהם למשפחה (כדי לא לאבד נתוני
  תלמידים בטעות), ומוחקת את אנשי הקשר לשעת חירום שהיו משויכים לאותה משפחה

- **שיבוץ מלמדים מלא**: יובאו שיבוצי בוקר/אחה"צ/עוזר לכל הכיתות מתוך קובץ השיבוץ
  שסופק (28 כיתות, 53 שיבוצים, כולל 23 מלמדים חדשים שלא היו קיימים קודם). הסניף
  (סוקולוב/נפחא/בן פתחיה) עודכן אוטומטית לכל כיתה מאותו קובץ
- **מיון עולה/יורד** בכל עמודה בטבלאות הראשיות (תלמידים, מלמדים, משפחות, כיתות) —
  לחיצה על כותרת העמודה ממיינת, לחיצה נוספת הופכת כיוון
- **חיוג מהיר**: כל מספר טלפון המוצג במערכת הוא קישור לחיצה שמעתיק את המספר
  ללוח (Ctrl+C) אוטומטית. **הערה חשובה**: מטעמי אבטחה, דפדפנים אינם יכולים לשלוח
  לחיצות מקלדת (F7, Enter) לתוכנה חיצונית ברמת מערכת ההפעלה — זהו חסם טכני
  שאי אפשר לעקוף מתוך דף אינטרנט. לכן השלב של F7 ואז Enter בתוכנת החיוג עדיין
  ידני, אבל המספר כבר מועתק אוטומטית ומוכן (כך שנשארות רק 2 לחיצות מקלדת
  במקום הקלדת המספר). אם תוכנת החיוג שלכם תומכת בקישורי `tel:` כברירת מחדל
  במערכת ההפעלה, הלחיצה תפעיל אותה ישירות ללא צורך אפילו ב-F7
- **הצהרת בריאות תלמיד**: דוח חדש (תחת "דוחות") שמשחזר את טופס הצהרת ההורים
  המקורי, עם **מילוי אוטומטי מלא** של כל השדות העובדתיים (פרטי תלמיד, הורים,
  כתובת, איש קשר לחירום, ותיבות הסימון של מצב הבריאות/תרופות/אלרגיות לפי
  הנתונים בפועל). ניתן לבחור כיתה אחת, כמה כיתות, או את כל הכיתות, וכל הטפסים
  נפתחים כעמוד הדפסה אחד (כל תלמיד בעמוד נפרד) — בחלון ההדפסה ניתן לבחור
  "שמירה כ-PDF" לייצוא קובץ PDF אמיתי. שדות חתימה ותאריך נשארים ריקים
  לחתימה פיזית של ההורים, כמובן
